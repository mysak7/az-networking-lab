How Private DNS Resolution Works

nslookup <storage>.blob.core.windows.net from inside the VM

1

VM queries Azure DNS (168.63.129.16) for mystorageacct.blob.core.windows.net.

2

Azure DNS sees a CNAME: mystorageacct.blob.core.windows.net → mystorageacct.privatelink.blob.core.windows.net.

3

The Private DNS Zone privatelink.blob.core.windows.net is linked to this VNet via VNet Link, so Azure DNS consults it.

4

The zone has an A record (auto-registered by the private endpoint): mystorageacct → 10.0.2.x. Returns the private IP.

5

VM opens a TCP connection to 10.0.2.x — the PE NIC inside the PE Subnet. Traffic never leaves the VNet. Storage's public endpoint is unreachable.
