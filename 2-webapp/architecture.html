Request Flow

1
Client → App Gateway WAF (HTTP :80)

Internet traffic hits the Standard Static Public IP. The NSG on appgw-subnet allows ports 80, 443, and GatewayManager 65200–65535. WAF inspects the request with OWASP 3.2 rules in Prevention mode — malicious requests are blocked before they reach the app tier.

2
App Gateway → Frontend App Service (HTTPS :443)

App Gateway forwards traffic to the frontend App Service hostname over HTTPS. pick_host_name_from_backend_address = true ensures correct SNI for App Service. The frontend enforces an IP restriction allowing only the appgw-subnet service endpoint — direct internet access is denied.

3
Frontend → Backend API (HTTPS, via VNet Integration)

The frontend calls the backend using the BACKEND_URL app setting. Outbound traffic from the frontend App Service egresses through frontend-subnet (10.0.2.0/24) via Regional VNet Integration. The backend enforces an IP restriction allowing only this subnet — no public access.

4
Backend → Key Vault (private endpoint, HTTPS)

At startup the backend resolves storage-connection-string via a Key Vault reference (@Microsoft.KeyVault(SecretUri=...)). The private DNS zone privatelink.vaultcore.azure.net resolves the vault FQDN to the private endpoint NIC in pe-subnet. The backend's system-assigned managed identity is granted Get / List via an access policy.

5
Backend → Storage Account (private endpoint)

The storage account has default_action = "Deny" — all public access is blocked. The backend egresses through backend-subnet into the private endpoint in pe-subnet. The private DNS zone privatelink.blob.core.windows.net routes blob traffic privately. The managed identity also holds Storage Blob Data Contributor for passwordless RBAC access.

Subnet Layout

appgw-subnet

10.0.1.0/24

App Gateway WAF v2. Service endpoint Microsoft.Web enables App Service IP restrictions to trust VNet-origin traffic. NSG attached (Allow 80, 443, GatewayMgr).

frontend-subnet

10.0.2.0/24

Frontend App Service outbound via Regional VNet Integration. Delegated to Microsoft.Web/serverFarms. Outbound IPs used as allow-list source on backend.

backend-subnet

10.0.3.0/24

Backend App Service outbound via Regional VNet Integration. Delegated to Microsoft.Web/serverFarms. Outbound IPs reach pe-subnet private endpoints.

pe-subnet

10.0.4.0/24

Private endpoint NICs for Storage Account (blob) and Key Vault. Network policies disabled — required for private endpoints. Linked to both private DNS zones.

Identity & Access

Frontend App Service — System Assigned Identity

  • No direct Azure RBAC roles assigned
  • Calls backend over HTTPS within VNet
  • Identity used for future integrations (e.g. Key Vault, ACR)

Backend App Service — System Assigned Identity

  • Key Vault Access Policy: Get, List on secrets
  • RBAC: Storage Blob Data Contributor on storage account
  • Resolves KV references at App Service runtime — no secrets in code

Terraform Deployer (Service Principal / User)

  • Key Vault Access Policy: Get, Set, List, Delete, Purge, Recover
  • Required to write storage-connection-string secret during terraform apply
  • Should be removed / scoped down in production CI pipelines

Private DNS Zones

  • privatelink.blob.core.windows.net → linked to VNet
  • privatelink.vaultcore.azure.net → linked to VNet
  • Auto-registration disabled; A records managed by private endpoint group

Security Controls Summary

Control Where Applied Detail
WAF (OWASP 3.2 Prevention) App Gateway Blocks SQLi, XSS, RFI, LFI, and other OWASP Top 10 exploits before they reach the app tier. Request body check enabled, 128 KB body limit, 100 MB upload limit.
IP Restrictions Frontend & Backend App Services Default action = Deny. Frontend allows only appgw-subnet (service endpoint). Backend allows only 10.0.2.0/24 (frontend VNet integration outbound).
Regional VNet Integration Frontend & Backend App Services All outbound traffic from App Services routes through dedicated delegated subnets, enabling private connectivity to storage and Key Vault endpoints.
Private Endpoints Storage Account, Key Vault Storage and Key Vault are not reachable from the public internet. Private DNS zones resolve FQDNs to NIC IPs in pe-subnet.
Managed Identity + Key Vault References Backend App Service Connection string never stored in code or environment variables as plaintext. Azure resolves @Microsoft.KeyVault(...) at runtime using the app's identity.
TLS 1.2 Minimum Frontend & Backend App Services minimum_tls_version = "1.2" and https_only = true enforced on both app services.
Storage Public Access Deny Storage Account default_action = "Deny" in network rules. Only traffic via private endpoint reaches blob storage. AzureServices bypass allows platform-level operations.
Soft Delete Key Vault 7-day soft delete retention. Purge protection disabled for lab — enable in production to prevent accidental secret destruction.
