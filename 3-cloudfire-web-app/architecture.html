What to set in Cloudflare dashboard

  1. 1 Go to cloudflare.com → Add a site → enter mysak.fun
  2. 2 Select Free plan
  3. 3 Cloudflare scans existing Azure DNS records → review & import
  4. 4 Copy the 2 Cloudflare nameservers shown (e.g. arnold.ns.cloudflare.com)
  5. 5 In Azure DNS Zone for mysak.fun → update NS records to Cloudflare's nameservers
  6. 6 Run terraform apply → CNAME + TXT records created automatically
  7. 7 Get your Zone ID from Cloudflare dashboard → right side panel → paste in terraform.tfvars

Security: Cloudflare IP Lock

Why
Without it, anyone can bypass Cloudflare by hitting azurewebsites.net directly
How
22 Cloudflare IP ranges (IPv4 + IPv6) allowed; default action = Deny
Management plane
SCM endpoint (Kudu) has separate, open restrictions — Azure management APIs are unaffected
IP range updates
Cloudflare rarely changes IPs; update the list in main.tf locals if needed
Verify lock
Try curl on azurewebsites.net URL directly — should get 403
